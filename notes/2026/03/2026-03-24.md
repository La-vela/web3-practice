### 2. 블록체인 데이터 구조 (Blockchain Data Structures)
- 다양한 유형의 탈중앙화 애플리케이션(dApp) 데이터를 전 세계적으로 저장하는 분산 데이터베이스와 같다.
- 각 블록은 블록 헤더(Header)와 본문(Body)의 두 부분으로 구성

[블록 헤더]
    - parentHash: 이전 블록의 해시 값을 기록합니다.
    - 타임스탬프: 블록이 생성된 정확한 시간을 기록합니다.
    - nonce: 작업증명(PoW) 채굴 과정에 사용됩니다.
    - 난이도: 채굴 난이도를 나타냅니다.
    - coinBase: 채굴자의 계정 주소를 식별합니다.
블록 헤더에는 매우 중요한 해시 값 아래의 세 가지가 기록
    - StateRoot: 블록체인 상태 트리의 루트 해시값 (잔액, 계약 코드 등 계정으 ㅣ상태 정보가 기록)
    - reciptRoot: 거래 실행 결과
    - transactionRoot: 거래 트리의 루트 해시 값

[블록 본체]
    - nonce: 송신자 계정의 거래 카운터로, 해당 계정이 이 블록체인에서 수행한 총 거래 횟수를 나타냅니다.
    - gasPrice: 송신자가 가스 단위당 지불하고자 하는 가격.
    - gasLimit: 송신자가 이 거래를 위해 소비할 최대 가스량입니다.
    - 받는 사람: 수신자의 계좌 주소.
    - 값: 전송할 이더리움의 양.
    - 데이터: 스마트 계약 관련 바이트코드.
    - v, r, s: 발신자의 개인 키로 생성된 거래 서명입니다.

[머클 패트리샤 트리]
머클 트리의 리프 노드, 압축 접두사를 활용한 패트리샤 트리의 장점을 결합한 트리

### 3. 해시 함수 및 블록체인 연동(Hash Functions and Blockchain Linkage)
- 1950년 해시 테이블과 같은 데이터 구조에서 데이터의 빠른 검색 및 추출을 위해 사용
MD4 ->MD5 ->SHA0 ->SHA1 ->SHA2 ->SHA3
현재 가장 발전된 표준은 벨기에 암호학자들이 설계하고 2015년에 NIST 표준이 된 케착 알고리즘을 기반으로 하는 SHA-3이다.

- 결론적으로 해시는 거래 데이터의 무결성과 불변성을 보장하고, 블록들을 서로 연결하여 블록체인을 형성, 작업증명 매커니즘을 구현하여 계산 퍼즐 해결하고 보안을 강화한다.

### 4. 비대칭 암호화 및 공개-개인 키 시스템(Asymmetric Encryption and Public-Private Key Systems)

- 공개 키와 개인 키는 반드시 쌍으로 사용해야 하며, 공개 키는 공개적으로 배포될 수 있으므로 누구나 이를 사용하며 정보를 암호활 수 있지만, 개인 키를 가진 사람만이 해당 종보를 복화할 수 었다. 따라서 공개 키가 공개되어 있더라도 개인 키가 없으면 누구도 암호화된 정보이 내용을 읽을 수 없다.
- 공개-개인 키 시스템: 비대칭 암호화 기반으로 대칭 암호화는 빠르지만 키 전송 측면에서 위험성이 높고, 비대칭 암호화는 키 배포를 보호하는 데 더 안전한 솔루션을 제공
- 메시지 → 해시 → 개인키로 서명 / 개인키는 오직 발신자만 가지고 있음 / 공개키로 검증 성공하면 그 개인키 주인이 보낸 것

### 6. 작업증명(PoW)
- 채굴자의 임무는 논스(Nonce) 값을 0부터 시작하여 지속적으로 변경하고, 난이도 목표값 이하의 해시값이 나올 때까지 블록 헤더의 해시값을 다시 계산하는 것입니다. 채굴자는 다양한 논스 값을 시도합니다.
- 예) 해시 값 세 자리 숫자가 0으로 시작하는 값을 찾아내는 것(성공을 의미)
- 최장 사슬 원리: 가장 긴 체인이 네트워크에서 수행된 가장 중요한 작업량을 나타내기 때문에 가장 긴 체인의 원칙을 따른다.(채굴자들의 채굴 원칙)

결론적으로 51% 차지하는 것이 적용 원칙

### 7. 지분증명(PoS)
- 지분증명 매커니즘은 네트워크 운영에 필요한 에너지 소비를 크게 줄이고, 거래 수수료나 블록 보상과 같은 경제적 인센티브를 통해 코인 보유자들이 네트워크 활동에 참여하도록 장려 (말 그대로 지분을 많이 가진 자들이 활동을 많이 하도록 하는 정책)

### 8. 참고 (Other Consensus Algorithms)
블록체인 기술에서는 잘 알려진 작업증명(PoW)과 지분증명(PoS) 합의 메커니즘 외에도 다양한 합의 알고리즘이 사용되고 있습니다.다음은 몇 가지 일반적인 합의 알고리즘입니다.

- 이력 증명(PoH) : PoH는 이벤트의 순서와 시간을 검증하기 위한 신뢰할 수 있는 시간 기록을 생성하는 것을 목표로 하며, 네트워크 참여자가 서로 통신할 필요 없이 이벤트 발생 순서를 검증할 수 있도록 합니다.
- 실용적 비잔틴 장애 허용(PBFT) : 시스템 내에 일정 비율의 악의적인 노드가 존재하더라도 이를 허용하여 네트워크 전체에서 합의가 이루어지도록 설계되었습니다. PBFT는 특히 프라이빗 블록체인이나 컨소시엄 블록체인에 적합합니다.
- 위임형 지분증명(DPoS) : 이는 지분증명(PoS)의 개선된 버전으로, 사용자들이 투표권을 소수의 대표자에게 위임하여 이들이 블록체인에 새로운 블록을 검증하고 추가하는 역할을 담당합니다. DPoS는 네트워크 확장성과 거래 속도를 향상시키는 것을 목표로 합니다.
- 권한 증명(PoA) : PoA 방식에서는 사전에 선정된 신뢰할 수 있는 노드(검증자)가 트랜잭션과 블록의 유효성을 검증합니다. 이러한 노드들은 공개적인 신원을 통해 책임을 지며, 속도와 효율성이 요구되는 네트워크에 적합합니다
- 순수 지분증명(Pure PoS) : PPoS는 검증자를 무작위로 선택함으로써 네트워크의 탈중앙화와 보안을 보장하여 지분증명(PoS)의 잠재적인 중앙 집중화 문제를 해결합니다.
- 공간 증명(PoSpace) 또는 용량 증명(PoC) : 이 알고리즘들은 네트워크 참여자들이 제공하는 저장 공간의 양을 기준으로 블록 생산자를 선택합니다. 이 알고리즘의 장점은 에너지 소비가 적다는 것입니다.
- 인증 증명(PoAuth) : PoA와 유사하지만, PoAuth는 검증자가 공개된 신원을 보유해야 하며, 일반적으로 커뮤니티에서 선정한 신뢰할 수 있는 개인이나 기업입니다.

## 오답노트
- 이더리움 송금에는 잔액이 포함되지 않는다. 잔액은 트랜잭션(Transaction)에 있는 게 아니라  State Tree (상태 트리)에 저장됨
- 거래 해시는 서명을 포함한 해시고, 서명 전 해시는 거래 데이터만 포함
    - 순서
        1. 거래 데이터 생성
        2. 서명 전 해시 생성
        3. 개인키로 서명 → (v, r, s)
        4. 최종 거래 생성
        5. 거래 해시 계산 (서명 포함)
    - [!] 결론적으로 이더리움에서 거래 해시는 거래 데이터와 서명을 모두 기반으로 계산된다는 내용은 틀림
