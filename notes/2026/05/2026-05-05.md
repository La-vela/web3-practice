## Learn

### assert
해당 조건을 참으로 가정하고 절대 거짓이 되어서는 안 되는 내부 오류 및 제약 조건을 검사하는 것에 사용
- 예를 들어 항상 양수가 나와야 한다면 'assert result >= 0' 수식을 사용하여 검사
- assert로 인해 에러가 발생하면, 기존의 가스는 물론이고 assert문의 가스 사용비도 생긴다.
- 다만, 해당 트랜잭션에 변경된 데이터는 Rollback되며 원상복구(Revert)된다.

```
assert(balance[msg.user] < totalSupply);
```

### try catch
try-catch문은 비슷
- 자바랑 차이점이라면 내 코드 안에서 assert나 require가 실패하면 트랜잭션 전체가 Rollback되어 try-catch가 내 코드 내부의 에러를 잡아서 회복할 수 없다는 단점이 있다.

add(num1,num2)에 대한 오류 처리 결과로 정수형 합계를 반환
```
try add(num1,num2) returns(uint sum) {

} catch {

}
```

- catchError(...){...}문에서는 외부에서 발생한 require 또는 revert문으로 인해 발생하는 오류를 처리할 수있다.
- require로 발생ㄹ한 에러는 catch Error(string memory reason)로 잡아야함
