## Guide Project 2


### minting (민팅)
mint란 새 토큰을 새로 발행해서 특정 주소에 넣는 것을 말함

1. mapping(계좌별 잔액표) 장부 역할을 하는 저장 구조를 선언
2. 호출한 사람이 owner(소유자)인지 확인
3. 대상 주소의 잔액과 총 공급량을 업데이트

- 매핑 private balances; 지갑별 잔액 관리 장부
- uint256 public totalSupply; 현재 유통 중인 토큰의 총량을 저장
- address private owner; 주인 주소를 저장하는 변수 (민팅 같은 권한이 필요한 함수를 만들 때 이사람만 실행하도록 사용됨)

4. mint function (발행함수) 정의
5. 발신자의 주소가 소유자 로 등록된 발급기관의 주소 와 일치하는지 확인 -> 일치하지 않으면 실패(require)
6. 잔액 에 기록된 잔액을 업데이트 (수신자의 계정에 발행된 토큰 수량을 추가)
7. 총 공급량 에 발행량을 추가(새로 반영하기 위함)

```
// SPDX-License-Identifier: MIT
pragma solidity 0.8.17;

contract MyToken {
    //mapping used to store the balance corresponding to each address
    mapping(address => uint256) private balances;

    //A uint256 variable is used to store the total supply of the token. It is defined as public and can be queried by anyone.
    uint256 public totalSupply;
    
    //An address variable is used to store the issuer of this token. This is used for some permission control
    address private owner;

    // 4.
    constructor() {
        owner = msg.sender;
    }

    // 5. function used to mint tokens
    function mint(address recipient, uint256 amount) public {

        require(msg.sender == owner, "Only the owner can perform this action");
     }
     // 6.
    balances[recipient] += amount;

    // 7.
    totalSupply += amount;
}
```
