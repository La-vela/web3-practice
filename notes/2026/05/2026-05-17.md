# 1. Web3 / 블록체인 기초

## Web의 진화

### Web 1.0 (1990년대 ~ 2004년경)
- 읽기 중심(Read-only)
- 정적 웹페이지
- 사용자 참여 제한적
ex) yahoo, 개인 html 홈페이지

### Web 2.0 (2004년 ~ 현재)
- 읽기 + 쓰기(read & write)
- SNS, 플랫폼 중심
- 사용자 생성 콘텐츠 활성화
- 데이터가 기업 서버에 집중
ex) YouTube, Facebook, Instagram

### Web 3.0 (2014년 ~ 현재/미래)
- 읽기 + 쓰기 + 소유(Read, Write, Own)
- 탈중앙화
- 블록체인 기반
- 사용자 데이터 / 자산 소유권 가조
- 스마트 컨트랙트 활용
ex) Ethereum, NFT, DAO, DeFi 서비스

## Ethereum 핵심

### Ethereum (이더리움)
- 스마트 컨트랙트를 실행하는 블록체인 플랫폼
- 비탈릭 부테란(Vitalik Buterin)이 제안
- DApp(탈중앙화 앱) 실행 가능

### Smart Contract (스마트 컨트랙트)
- 블록체인 위에서 실행되는 프로그램
- 조건 충족 시 자동 실행
- 변경이 어렵고 투명한 실체

## PoW vs PoS
이더리움은 작업 증명 방식을 채택

### Proof of work (작업 증명)
- 작업량 기반, mining(채굴)방식
- 높은 전력 사용

### Proof of Stake
- 지분 기반, 코인을 많이 스테이킹한 검증자가 참여
- 에너지 효율적

# 3. Function (함수)

## 기본 함수

```solidity
function add(uint a, uint b) public pure returns(uint) {
    return a + b;
}
```

## 함수 가시성(Visibility)
- public: 어디서든 호출 가능
- private: 현재 컨트랙트 내부에서만 사용
- internal: 상속된 컨트랙트 포함 내부 사용
- external: 외부 호출 전용

## 상태변경
pure나 view는 데이터를 수정하지 않으니 외부에서 그냥 조회만 할 때는 가스비가 공짜
- pure: 블록체인 상태에 접근하지 않는다는 동작
- view: 블록체인 상태에 접근해 읽기만 하는 동작
