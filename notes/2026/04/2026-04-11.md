## 개념

### Mapping
- uint (키): ID 번호 키로 사용
- => (화살표): "매핑된다"는 뜻
- uint (값): 다른 ID 번호를 값으로 저장
- IDToID (이름): 매핑의 이름

### 목적
보통 은행은 중개자, 은행원은 수수료를 내고, 은행원이 장부를 조작할까봐 걱정함

스마트 컨트랙트의 주요 목표는 은행원(중가재)이 없다. 그래서 코드(솔리디티)가 은행원으로 생각하고, 코드의 신뢰성으로 탈중앙화(DeFi) 방식이다.


## 코드 & 퀴즈


```
mapping(uint => uint) IDToID; // ID끼리 매핑 (uint 키 -> uint 값)

mapping(address => uint) public pool; // 지갑 <-> 풀 잔액 매핑 주소 키 -> 정수 값, public으로 getter 자동생성
```
