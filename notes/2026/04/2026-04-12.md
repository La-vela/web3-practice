## 개념
Solidity 103은 고급 기능과 보안에 관한 사항으로 실전 배포 단계
Checks-Effects-Interactions 패턴
- 이미 검증된 라이브러리인 'OpenZeppelin'을 활용
- AccessControl: 누가 이 함수를 실행할 수 있는지
- Reentercyguard: 재진입 방지용 '문지기 함수'
- Pausable: 긴급 상황에 컨트랙트의 모든 기능을 정지시키는 단계

## 코드 & 퀴즈


``` 가스 최적화 기법
contract ImmutableExample {
    uint immutable MAXFEE;
  constructor(uint _maxFee) {
    MAXFEE = _maxFee;
  }
}
```

- immutable 타입의 uint 변수 MAXFEE를 선언하고 생성자에서 초기화하는 가스 최적화 스마트 컨트랙트
- 배포 시점에 결정되지만 고정된 값 (수수료 한도, 소유자 주소 등)을 immutable로 선언하면 10-20% 가스 절약이 됩니다.
